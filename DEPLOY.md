# Deploying your TanStack Start Project

This project is built using **TanStack Start**, a full-stack React framework powered by **Vite** and the **Nitro** server engine. Because it uses Nitro, it is extremely portable and can be deployed to almost any hosting provider (Cloudflare, Vercel, Netlify, self-hosted Node.js, etc.) with minimal configuration.

## Prerequisites

Before deploying, ensure you have:
1. Pushed your project to a Git repository (GitHub, GitLab, or Bitbucket).
2. Defined any required environment variables (e.g., API keys, database URLs) in your hosting provider's dashboard.

---

## Option 1: Cloudflare Pages (Recommended & Default Target)

The project's build setup (`@lovable.dev/vite-tanstack-config`) is pre-configured to build for **Cloudflare Pages** by default.

### Automated Deployment (Git Integration)
1. Log in to the [Cloudflare Dashboard](https://dash.cloudflare.com/).
2. Go to **Workers & Pages** > **Create application** > **Pages** > **Connect to Git**.
3. Select your repository.
4. Set up the build settings:
   - **Framework Preset**: None (or select Vite if available, but configure custom commands below).
   - **Build Command**: `npm run build` (or `bun run build` if using Bun).
   - **Build output directory**: `.output/public`
5. Click **Save and Deploy**.

### Manual Deployment (Wrangler CLI)
If you want to deploy directly from your local terminal:
1. Install Wrangler globally or use npx:
   ```bash
   npx wrangler pages deploy .output/public
   ```
2. Follow the terminal prompts to log in and select/create a project.

## Option 2: Vercel

Vercel is an excellent option for serverless hosting. Nitro will automatically detect the Vercel environment and build the project using Vercel's Serverless Functions.

### Automated Deployment (Git Integration)
1. Go to the [Vercel Dashboard](https://vercel.com/) and click **Add New** > **Project**.
2. Import your Git repository.
3. Vercel should auto-detect the framework. If not, customize the settings:
   - **Build Command**: `npm run build`
   - **Output Directory**: `.vercel/output` (Vercel automatically detects the Nitro build structure).
4. Click **Deploy**.

---

## Option 3: Netlify

Netlify supports serverless deployment natively. Nitro automatically configures Netlify Functions during the build step.

### Automated Deployment (Git Integration)
1. Log in to the [Netlify Dashboard](https://app.netlify.com/) and click **Add new site** > **Import an existing project**.
2. Select your repository.
3. Configure the build settings:
   - **Build Command**: `npm run build`
   - **Publish directory**: `.netlify/images` or let Netlify auto-detect the output structure generated by Nitro under `.netlify`.
4. Click **Deploy Site**.

---

## Option 4: Railway (Recommended for Server Hosting)

Railway is a cloud platform that makes it extremely easy to host dynamic Node.js servers.

### Automated Deployment (Git Integration)
1. Go to [Railway](https://railway.app/) and sign in.
2. Click **New Project** > **Deploy from GitHub repo**.
3. Select your repository `Hemanth-Techi404/Automiq-site`.
4. Once added, go to the **Variables** tab of the service and click **Add Variable**:
   - Add `NITRO_PRESET` with the value `node-server` (This tells Nitro to build a Node.js server instead of Cloudflare Pages).
   - Add `PORT` with the value `3000` (or leave it blank so Railway assigns one automatically).
   - Add `OPENAI_API_KEY` with your OpenAI API key.
   - Add `SARVAM_API_KEY` with your Sarvam API key.
5. Railway will automatically find the `start` script in `package.json` (`node .output/server/index.mjs`) and use it to run your server.
6. Under **Settings** > **Public Networking**, click **Generate Domain** to get a public URL for your site.

## Option 5: Self-Hosted Node.js Server (Docker / VPS / Render / Other)

If you want to run a persistent Node.js server (such as on a Virtual Private Server (VPS), Render, or inside Docker) manually:

### 1. Build the Project
Run the build command with the `NITRO_PRESET` environment variable set to `node-server`:
- **Windows (PowerShell)**:
  ```powershell
  $env:NITRO_PRESET="node-server"; npm run build
  ```
- **macOS / Linux / Bash**:
  ```bash
  NITRO_PRESET=node-server npm run build
  ```

This will generate a standalone Node.js server inside the `.output` directory:
- `.output/server/index.mjs` (The server entry point)
- `.output/public/` (Static client assets)

### 2. Run the Server
To run the server locally or on your VPS, run:
```bash
node .output/server/index.mjs
```
The server will start (default port: `3000`). You can configure the port and host using environment variables:
```bash
PORT=8080 HOST=0.0.0.0 node .output/server/index.mjs
```

### 3. Docker deployment
You can containerize this application using a `Dockerfile`:

```dockerfile
# Build stage
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
ENV NITRO_PRESET=node-server
RUN npm run build

# Production stage
FROM node:20-alpine
WORKDIR /app
COPY --from=builder /app/.output ./.output
ENV PORT=3000
ENV NODE_ENV=production
EXPOSE 3000
CMD ["node", ".output/server/index.mjs"]
```

## Environment Variables

When deploying to production:
- Ensure any public variables starting with `VITE_` are defined in your deployment dashboard, as Vite injects these at build time.
- Server-side environment variables (database credentials, API keys) can be set directly in your hosting provider's dashboard and will be read at runtime.
